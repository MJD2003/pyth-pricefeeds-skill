# Pyth Price Feed IDs

Feed IDs are 32-byte hex identifiers unique to each price feed. The same feed ID works across all chains.

**Full list:** https://docs.pyth.network/price-feeds/price-feeds

> Tip: Use `Stable` feeds for production. `Beta` feeds may have less coverage.

## How to Find Any Feed ID

1. Visit https://docs.pyth.network/price-feeds/price-feeds
2. Search by asset name (e.g., "BTC", "AAPL", "EUR")
3. Copy the `Stable` feed ID (not `Beta`)
4. Use with `0x` prefix in Solidity: `bytes32 feedId = 0xff61...`
5. Use without prefix in Hermes API: `ids[]=ff61...`

## Dynamic Feed Discovery (Recommended)

**You do NOT need to hardcode feed IDs.** Fetch all available feeds programmatically at runtime from the Hermes API:

**Endpoint:** `https://hermes.pyth.network/v2/price_feeds`
**Docs:** https://hermes.pyth.network/docs/#/rest/price_feeds_metadata

### Fetch ALL Feeds

```bash
# Get every available Pyth price feed (1000+)
curl "https://hermes.pyth.network/v2/price_feeds"
```

### Filter by Asset Type

```bash
# Crypto feeds only
curl "https://hermes.pyth.network/v2/price_feeds?asset_type=crypto"

# Equity feeds only
curl "https://hermes.pyth.network/v2/price_feeds?asset_type=equity"

# FX feeds only
curl "https://hermes.pyth.network/v2/price_feeds?asset_type=fx"

# Metal feeds
curl "https://hermes.pyth.network/v2/price_feeds?asset_type=metal"
```

### Search by Name

```bash
# Search for BTC-related feeds
curl "https://hermes.pyth.network/v2/price_feeds?query=btc"

# Search for Apple stock
curl "https://hermes.pyth.network/v2/price_feeds?query=aapl&asset_type=equity"
```

### JavaScript — Dynamic Resolution

```javascript
// Fetch all feeds at once
const response = await fetch("https://hermes.pyth.network/v2/price_feeds");
const feeds = await response.json();

// Each feed has id + attributes (symbol, asset_type, base, quote_currency)
feeds.forEach(feed => {
  console.log(`0x${feed.id}`, feed.attributes.symbol);
});

// Resolve a specific symbol to its feed ID
function resolveFeedId(feeds, symbol) {
  const normalized = symbol.toUpperCase();
  const match = feeds.find(f => {
    const s = (f.attributes?.symbol || "").toUpperCase();
    return s === normalized || s.endsWith(`.${normalized}`);
  });
  return match ? "0x" + match.id : null;
}

const ethFeedId = resolveFeedId(feeds, "ETH/USD");
// "0xff61491a931112ddf1bd8147cd1b641375f79f5825126d665480874634fd0ace"
```

### Python — Dynamic Resolution

```python
import httpx

# Fetch all feeds
feeds = httpx.get("https://hermes.pyth.network/v2/price_feeds").json()
print(f"Total feeds: {len(feeds)}")

# Filter by asset type
crypto = httpx.get("https://hermes.pyth.network/v2/price_feeds?asset_type=crypto").json()
print(f"Crypto feeds: {len(crypto)}")

# Resolve a symbol
def resolve_feed_id(symbol):
    feeds = httpx.get(f"https://hermes.pyth.network/v2/price_feeds?query={symbol.replace('/', '')}").json()
    normalized = symbol.upper()
    for f in feeds:
        s = (f.get("attributes", {}).get("symbol", "") or "").upper()
        if s == normalized or s.endswith(f".{normalized}"):
            return "0x" + f["id"]
    return None
```

### Response Format

Each feed in the response has:
```json
{
  "id": "ff61491a931112ddf1bd8147cd1b641375f79f5825126d665480874634fd0ace",
  "attributes": {
    "symbol": "Crypto.ETH/USD",
    "asset_type": "crypto",
    "base": "ETH",
    "quote_currency": "USD",
    "description": "ETH/USD",
    "generic_symbol": "ETHUSD",
    "weekly_schedule": "..."
  }
}
```

### When to Use Dynamic vs Hardcoded

| Approach | When to Use |
|----------|-------------|
| **Dynamic** (`/v2/price_feeds`) | Apps with user-selectable assets, dashboards, search UIs |
| **Hardcoded** (table above) | Contracts with fixed feeds, known asset pairs |
| **Build-time cache** | Fetch once at build, embed in config |

> See `assets/typescript/feed-discovery.ts` for a full TypeScript module with caching, batch resolution, and registry building.
